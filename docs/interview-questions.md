# Git Interview Questions

1. Explain the difference between working tree, staging area, and repository.
2. What is the difference between `git fetch` and `git pull`?
3. When would you use merge instead of rebase?
4. What is a fast-forward merge?
5. What does `git pull --rebase` do?
6. Why is `git push --force-with-lease` safer than `git push --force`?
7. Explain `git reset --soft`, `git reset --mixed`, and `git reset --hard`.
8. When should you use `git revert` instead of `git reset`?
9. How do you recover a deleted branch?
10. How does `git bisect` identify a regression?
11. What is the difference between an annotated tag and a lightweight tag?
12. What is `origin/main` and how is it different from local `main`?
13. How do CODEOWNERS and required status checks improve team workflow?
14. What is the golden rule of rebase?
15. How can `git rerere` help repeated conflict resolution?
